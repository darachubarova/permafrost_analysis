### ГЛАВА ВТОРАЯ. МЕТОДОЛОГИЧЕСКИЙ КОНТУР И ИНФОРМАЦИОННАЯ БАЗА ИССЛЕДОВАНИЯ

#### 2.1. Верификация источников данных и обоснование достоверности информационной базы

Для обеспечения научной достоверности и воспроизводимости результатов в работе использованы исключительно официальные ведомственные и международные базы данных:

1. **Гидрометклиматические данные:** Первичные суточные ряды температуры воздуха, атмосферных осадков и характеристик снежного покрова получены из фондов ФГБУ «Всероссийский научно-исследовательский институт гидрометеорологической информации – Мировой центр данных» (ФГБУ «ВНИИГМИ-МЦД», официальный портал [http://meteo.ru](http://meteo.ru)). Данный источник является опорным государственным архивом РФ, аккумулирующим валидированные бюллетени Всемирной метеорологической организации (WMO).
2. **Геокриологические данные (СТС/ALT):** Данные о глубине сезонного протаивания (Active Layer Thickness) получены из реестров международной программы циркумполярного мониторинга вечной мерзлоты CALM (Circumpolar Active Layer Monitoring), координируемой Университетом Джорджа Вашингтона (официальный депозитарий [https://www2.gwu.edu/~calm/](https://www2.gwu.edu/~calm/)). Данные верифицированы полевыми измерениями на специализированных геокриологических стационарах.
3. **Гидрологические данные:** Ряды ежедневных уровней воды (см над нулем поста) по гидропостам бассейна р. Лены получены из Автоматизированной информационной системы государственного водного реестра (АИС ГВР) и проверены по гидрологическим ежегодникам Государственного водного кадастра.
4. **Событийный архив чрезвычайных ситуаций:** Сведения о зафиксированных ЧС, опасных природных явлениях (ОЯ) и причиненном экономическом ущербе извлечены из официальных паспортов территорий и статистических отчетов Центра управления в кризисных ситуациях (ЦУКС) Главного управления МЧС России по Республике Саха (Якутия).

#### 2.2. Обоснование пространственно-временных границ расчетного контура

**Коррекция пространственного охвата:** Из исходного метеорологического массива полностью исключены станции Верхоянск и Оймякон. Несмотря на их климатическую значимость для региона, данные станции географически привязаны к бассейнам рек Яна и Индигирка соответственно. Их включение искажало бы Lead-Lag модели гидродинамического добегания и формирования паводковых аномалий непосредственно в русловой системе р. Лены. Финальный метеорологический контур сформирован по 4 репрезентативным станциям Ленского бассейна: **Киренск, Жиганск, Олёкминск, Якутск**.

**Обоснование расчетного периода (2008–2023 гг.):** Большинство используемых источников (метеорология, гидропосты) обладают сверхдлинными историческими рядами наблюдений (начиная с конца XIX века). Однако сквозной аналитический контур совместного анализа «Климат — Мерзлота — Гидрология — ЧС» жестко ограничен временным окном **2008–2023 гг.** по следующим критическим причинам:

1. *Доступность геокриологических данных:* В отличие от метеостанций, непрерывный сопоставимый мониторинг глубины сезонного протаивания грунта (СТС/ALT) на целевых площадках CALM вблизи рассматриваемых пунктов представлен только для периода 2008–2023 гг.
2. *Полнота ведомственной статистики МЧС:* Оцифрованные, детальные и верифицированные данные по локальным чрезвычайным ситуациям, ущербу (`damage_rub`) и превентивным затратам в ЦУКС МЧС приведены к единому стандарту и доступны без пропусков, начиная с 2008 года.

Таким образом, для метеорологических и гидрологических параметров весь избыточный исторический ряд отсекается, и расчет базовых статистик, аномалий и квантилей производится строго за период 2008–2023 гг. для обеспечения методологической однородности.

**Уровни временного агрегирования данных:** В соответствии со спецификой сопоставления разнородных процессов, в работе приняты два операционных уровня агрегирования: **суточный** (для краткосрочного гидрологического упреждения волны и фиксации экстремальных метеосигналов) и **годовой** (для инерционных трендов протаивания мерзлоты и межгодовых сценарных сопоставлений). Месячный уровень агрегирования исключен из конвейера обработки из-за высокой сглаженности пиковых паводковых угроз.

### Программные инструкции (Выполнить в репозитории для генерации Bar-Plots)

Для построения требуемых bar-plots межгодовой частоты аномалий запустите в вашем расчетном контуре следующий скрипт, который сохранит графики непосредственно для вставки в docx:

```python
import pandas as pd
import matplotlib.pyplot as plt

# Загрузка оперативной очищенной витрины за период 2008-2023 гг.
df = pd.read_csv("RESULTS (Read me, Dara)/data_tables/daily_anomalies_operational.csv")

# Агрегирование частоты встречаемости признаков (severity / аномалий) по годам
yearly_anomalies = df[df['severity'] != 'Normal'].groupby(['year', 'severity']).size().unstack(fill_value=0)

# Построение столбчатой диаграммы (Bar-plot)
plt.figure(figsize=(10, 6))
yearly_anomalies.plot(kind='bar', stacked=False, ax=plt.gca(), cmap='Reds')

plt.title('Распределение частоты опасных климатических аномалий по годам (2008–2023 гг.)', fontsize=12)
plt.xlabel('Год исследования', fontsize=10)
plt.ylabel('Количество суток с аномалиями', fontsize=10)
plt.grid(axis='y', linestyle='--', alpha=0.7)
plt.legend(title='Класс суровости (Severity)')
plt.tight_layout()

# Сохранение графика для вставки в диплом
plt.savefig("RESULTS (Read me, Dara)/plots/vkr_anomalies_yearly_bar.png", dpi=300)
plt.close()

```
